## 📱 2. PWA Architecture Upgrade

Arsitektur PWA dirancang ulang dari nol untuk mendukung **authentication-aware caching** dan **multi-tenant safety**.

### Service Worker (`sw.js`) — Rewrite Penuh

**Caching Strategy:**

| Endpoint | Strategy | Keterangan |
|----------|----------|------------|
| Static assets | Cache-First | Precache saat install |
| `GET /api/workspace\|tasks\|budget\|savings\|notes` | Network-First + IDB fallback | Offline support |
| `GET /api/auth/me` | Network-First | Session restore, no IDB |
| `POST/PUT/DELETE` | Network-only | Mutations tidak di-cache |
| `/api/auth/login\|register` | Never cached | Auth endpoints |
| `/api/admin/*` | Never cached | Admin endpoints |

**Multi-tenant Safety:**
- JWT di-decode (tanpa verify) hanya untuk extract `userId` sebagai cache key
- Semua data di-scope per `userId` — tidak ada cross-user leakage
- Logout message dari app → SW hapus data user tersebut dari IDB

**Message Handler:**
- `LOGOUT` — clear IDB data untuk userId tertentu
- `SKIP_WAITING` — aktivasi SW baru segera
- `REVALIDATE` — trigger background revalidation ke semua clients

### IndexedDB (`idb.js`) — Baru

Storage utama untuk offline data, scoped per user:

```
Database: wedspace-offline (v1)
Stores:
  workspace  → { userId, data, updatedAt }
  tasks      → { userId, data, updatedAt }
  budget     → { userId, data, updatedAt }
  savings    → { userId, data, updatedAt }
  notes      → { userId, data, updatedAt }
  meta       → { userId, data, updatedAt }
```

**API:**
- `idbGet(store, userId)` — baca data user
- `idbSet(store, userId, data)` — tulis data user
- `idbClearUser(userId)` — hapus semua data user (logout)
- `isStale(record, maxAgeMs)` — cek apakah data sudah stale

### PWA Utilities (`pwa.js`) — Baru

- `registerSW()` — register SW dengan auto-update check setiap 60 detik
- `notifySWLogout(userId)` — notify SW saat logout
- `notifySWRevalidate(userId)` — trigger revalidation
- `onSWMessage(callback)` — listen pesan dari SW
- `isOffline()` — cek status koneksi

### AppContext — Offline-Aware

- **Network-First** — selalu coba API dulu
- **IDB Fallback** — jika offline/network error, load dari IndexedDB
- **Auto-persist** — setiap response API disimpan ke IDB
- **Optimistic updates** — state update langsung, IDB update menyusul
- **Auto-revalidate** saat: window focus, kembali online, SW request

### AuthContext — Secure Logout

Logout flow yang aman:
1. Clear `localStorage` token
2. Reset React state
3. `idbClearUser(userId)` — hapus IDB data user
4. `notifySWLogout(userId)` — SW hapus data dari sisinya

### Offline Banner

Banner kuning muncul otomatis saat offline:
> "📡 Kamu sedang offline — menampilkan data terakhir yang tersimpan"

## 🏗️ 3. Infrastructure Improvements

### Root `package.json` — Diperbaiki

- **Hapus semua `dependencies`** — tidak perlu di root, sudah ada di `server/`
- `install:all` diubah: `npm install && npm run install:client` → `npm run install:server && npm run install:client`
- Tambah script `install:server` eksplisit

### Server `package.json` — Deploy Command

Tambah script `deploy` untuk Railway/production:

```bash
npm run deploy
# = setup-db → migrate → start server
```

Untuk Railway yang sudah punya PostgreSQL plugin:
```bash
npm run migrate && npm start
```

### Database Init — Admin Seed

`migrate.js` sekarang seed akun admin secara eksplisit:
- Baca dari env: `ADMIN_EMAIL`, `ADMIN_PASSWORD`
- Fallback: `admin@wedspace.id` / `Admin123456`
- Jika user sudah ada → update role ke `admin`
- Dipanggil sebelum seed demo data

### `.gitignore` — Diperbaiki

- `server/.env` — sebelumnya tidak ter-ignore (sudah di-untrack)
- `.kiro/` — format diperbaiki dari `.kiro/*` ke `.kiro/`
- Tambah `.env.*` pattern (kecuali `.env.example`)
- Tambah `*.db`, `*.sqlite` untuk database files

### Service Worker — Bug Fixes

- Filter `chrome-extension://` requests (tidak lagi error di console)
- Cache version bump ke `v3`
- Proper URL protocol check sebelum caching

### PWA Manifest — Diperbaiki

- Icon referensi diubah dari `.png` (tidak ada) ke `.svg` (ada)
- `background_color` dan `theme_color` diupdate sesuai design system baru
- Icon SVG diupdate dengan gold gradient
